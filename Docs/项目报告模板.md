## 二、项目创意与目标

### 2.1 为什么想做这个作品

> *说明你的动机。示例：*

随着虚拟现实技术的发展，传统的展览馆形式正在被重新定义。用户无需亲临现场，即可在虚拟空间中自由浏览展品、与展品进行深层次互动。本作品旨在探索 **VR 技术如何为文化教育传播提供新的载体**——让知识获取变得更直观、更具沉浸感。

### 2.2 希望为用户提供什么样的虚拟体验

> *说明设计目标。示例：*

本作品为用户提供以下核心体验：

1. **身临其境的观展感受**：通过完整3D场景和第一人称视角，让用户感觉确实走在一个真实的展览空间中。
2. **超越现实的交互方式**：用户可以拾取展品进行360度旋转观察、放大缩小——这在真实展览中无法实现。
3. **自由探索的乐趣**：没有固定路线，用户可以按自己的节奏和兴趣自由浏览各个展区。

## 三、系统设计与实现过程

### 3.1 系统概述

本系统是一个基于 Unity 引擎的 **第一人称虚拟展览馆**，用户通过键盘和鼠标与虚拟环境进行交互。

### 3.2 输入与输出

#### 输入（用户操作）
| 操作 | 系统响应 |
|------|----------|
| WASD 键 | 玩家在场景中前后左右移动 |
| 鼠标移动 | 控制视角旋转（水平360°+垂直限制80°） |
| E 键（靠近展品） | 触发展品交互——显示信息面板 |
| E 键（二次按） | 拾取展品到手中 |
| 鼠标左键拖拽 | 旋转手中的展品观察 |
| 鼠标滚轮 | 缩放手中的展品 |
| ESC 键 | 打开/关闭暂停菜单 |

#### 输出（系统反馈）
- **视觉反馈**：UI信息面板弹出、展品高亮、旋转动画
- **音频反馈**：背景音乐、展品专属音效、环境音
- **触觉反馈**：（如有VR设备可补充）

### 3.3 主要功能

1. **第一人称自由漫游** — 玩家可以在展览空间中自由行走
2. **展品信息查看** — 靠近展品后按E键查看详细介绍
3. **展品拾取与观察** — 将展品拿起，360度旋转、缩放查看细节
4. **区域导览系统** — 进入不同展区时自动触发介绍
5. **沉浸式音频** — 背景音乐+环境音+交互音效三层音频设计

### 3.4 实现过程

> *按步骤写清楚你怎么做的。示例：*

1. **环境搭建**：在Unity中创建新3D项目，搭建展览馆基础结构（地面+墙壁+天花板），使用ProBuilder或基础几何体拼合出展览空间。
2. **玩家系统**：挂载 FirstPersonController.cs 脚本实现 WASD 移动和鼠标视角控制。
3. **展品系统**：收集或制作3D模型，挂载 ExhibitBase/ExhibitPickup 脚本，配置展品名称、描述、交互距离等参数。
4. **UI系统**：使用Unity Canvas 制作开始面板、信息面板和交互提示文字。
5. **音频系统**：从免费音频网站下载素材，配置 AudioManager 实现BGM和SFX管理。
6. **测试与优化**：反复测试交互逻辑，修复碰撞、UI遮挡等问题。

## 四、3I 分析

### 4.1 Immersion 沉浸性

**我们的作品如何让用户产生进入虚拟环境的感觉？**

1. **完整的3D封闭空间**：展览馆是一个有天花、墙壁、地面的完整建筑内部，用户被"包围"在这个空间中。场景使用暖色调灯光和柔和的阴影，营造出真实的展览馆氛围。
2. **第一人称视角**：用户通过"自己的眼睛"观察世界，所见即所得——这是最直接的沉浸式体验方式。
3. **多层音频设计**：背景音乐建立起空间的情绪基调，环境音（空旷展馆的微弱回声）强化了"处于大型场馆"的感觉，交互音效则让每次操作都有实时回应。
4. **自由的视角控制**：用户可以任意方向观察环境，没有任何视角限制，如同真实的人在空间中转头观察。

### 4.2 Interaction 交互性

**用户可以如何与系统发生交互？**

1. **空间移动**：使用 WASD 在展览空间中自由行走，按 Shift 可以加速奔跑——用户的行为直接决定了观察的角度和顺序。
2. **靠近触发**：当用户走到展品附近（3米以内），系统会显示"按E查看详情"的提示——这模拟了真实展览中走近展品的行为。
3. **信息获取**：按E键弹出信息面板，展品自动旋转展示——这是真实展览无法做到的动态交互。
4. **拾取观察**（核心交互亮点）：用户可以"拿起"展品，用鼠标拖拽进行360度旋转、滚轮缩放——这在真实博物馆中是不可能的体验，数字展品突破了物理限制。
5. **区域感知**：进入不同展区时，系统自动播放区域介绍——模拟了真实展览中导览员讲解的效果。

**用户的操作会影响虚拟环境吗？**
- 是的。信息面板的显示/隐藏、展品的位置变化（被拿起），都直接源于用户的操作。
- 系统通过UI变化、音效播放和物体状态改变，对用户行为作出即时反馈。

### 4.3 Imagination 构想性/想象性

**我们的作品提供了什么新的体验、场景或创意？**

1. **超越物理限制的交互**：在真实博物馆中，展品被玻璃罩保护，观众只能远远观看。而在我们的虚拟展览中，用户可以"拿起"展品、旋转观察、缩放细节——这是对传统展览模式的颠覆。
2. **创造现实中难以实现的场景**：如果你选择了太空主题，用户就站在了"太空中"。这个场景在现实中需要极高成本甚至无法实现，但在虚拟空间中只需要几行代码。
3. **教育功能的延伸**：本作品不仅是"展览"，还可以扩展为"教育工具"——比如在每个展品上添加互动问答、小游戏，让学习变得更有趣。
4. **未来的应用想象**：这个作品展示了一个可能性——未来的教育、文化传播可能不再局限于物理空间。VR展览馆可以低成本复制、远程访问、支持多人同时参观互动。

### 5.2 收获与体会

> *示例：*

通过完成本作品，我们获得了以下收获：

1. **对VR系统3I特征有了实践层面的理解**：之前的学习停留在概念层面，实际开发后才真正体会到"沉浸感"是靠细节（光照、音频、视角、空间设计）堆出来的，不是"3D场景=沉浸"这么简单。
2. **掌握了Unity 3D开发的基本流程**：包括场景搭建、Collider触发、UI系统、音频管理、脚本通信等。
3. **体会到创意与成本的平衡**：在资源和技术有限的情况下，如何用最少的投入做出最好的效果，本身就是一次很好的工程实践。

### 5.3 不足与改进方向

1. **场景美术水平有限**：由于没有专业建模师，展品和环境的视觉精度还有提升空间。改进方向：未来可以引入更多Asset Store高质量资源，或与设计专业的同学合作。
2. **缺少多人互动功能**：目前是单人体验。改进方向：接入 Photon PUN 或 Mirror 网络框架，支持多人同时参观互动。
3. **VR设备适配尚未完成**：目前仅实现在PC上用键盘鼠标模拟。改进方向：接入 SteamVR / Oculus SDK，将第一人称视角映射到HMD，将移动映射到手柄。
4. **展品数量有限**：目前场景中只有6件展品。改进方向：设计展品加载系统，支持动态添加和远程更新展品内容。
