## 二、项目创意与目标

### 2.1 项目创意来源

本项目以“太空探索虚拟展览馆”为主题，原因是太空探索具有天然的沉浸感和想象空间。火箭、卫星、行星、空间站、月面基地、黑洞等内容在现实中难以让普通用户近距离接触，而虚拟现实和三维交互技术可以把这些遥远或抽象的对象转化为可以进入、观察和操作的虚拟展品。

传统展览通常以静态图片、文字说明和实物模型为主，用户更多是被动观看。本作品希望通过 Unity 搭建一个可运行的第一人称虚拟展馆，让用户能够在展馆中自由移动、选择参观路线、靠近展品并触发信息面板，从而获得更主动、更直观的科普体验。

### 2.2 项目目标

本项目希望实现以下目标：

1. **构建完整的虚拟展览空间**：包含地面、墙壁、天花板、展区、展台、导览路线和文字说明，使用户能够感受到自己进入了一个完整的展览馆。
2. **实现基本三维交互**：用户可以通过键盘和鼠标移动、观察、靠近展品、查看信息，并对部分展品进行拾取、旋转和缩放。
3. **体现 VR 系统 3I 特征**：通过第一人称漫游体现沉浸性，通过展品交互体现交互性，通过太空主题和未来场景体现构想性。
4. **满足展示视频要求**：作品设计了 1-5 导览路线、三大主题展区和 3I 分析展板，使最终视频可以自然展示 2-5 分钟。

## 三、系统设计与实现过程

### 3.1 系统概述

本系统是基于 Unity 引擎实现的第一人称虚拟展览馆。用户通过 `WASD` 键在展馆中移动，通过鼠标控制视角，通过 `E` 键与展品交互。展馆以太空探索为主题，设置了“火箭技术区”“行星探索区”“深空未来区”三个展区，并包含 9 个展示节点：长征火箭、人造卫星、火星、土星、国际空间站、深空探测器、月面基地沙盘、太阳系轨道模型和黑洞引力井。

当前作品在普通 Windows PC 上运行，使用键盘鼠标模拟虚拟现实体验。如果后续接入 VR 设备，可以将第一人称摄像机映射到头显视角，将 `E` 键交互映射到手柄扳机键或射线选择，将键盘移动映射到手柄摇杆移动。

### 3.2 输入与输出

#### 输入（用户操作）

| 操作 | 系统响应 |
|------|----------|
| `WASD` | 用户在展馆内前后左右移动 |
| `Shift` | 加速移动，方便快速浏览展区 |
| 鼠标移动 | 控制第一人称视角旋转 |
| 靠近展品后按 `E` | 打开展品信息面板 |
| 可拾取展品附近按 `E` | 将展品拿到镜头前观察 |
| 鼠标左键拖动 | 旋转手中展品 |
| 鼠标滚轮 | 缩放手中展品或调整观察距离 |
| `Esc` / 鼠标点击 / `Space` / `Enter` | 关闭信息面板或切换暂停状态 |

#### 输出（系统反馈）

- **视觉反馈**：展馆空间、展区标识、展品模型、地面导览路线、3I 分析展板、信息面板、开始界面和暂停界面。
- **交互反馈**：靠近展品时显示交互提示；按 `E` 后弹出展品信息；可拾取展品会移动到用户视角前方。
- **音频反馈**：使用 `maksymmalko-space-space-galaxy-universe-music-301239.mp3` 作为背景音乐，在开始参观后播放，增强展馆氛围。

### 3.3 主要功能

1. **第一人称自由漫游**  
   用户可以在展馆内自由行走和观察，参观路线由用户自主决定。

2. **三大主题展区**  
   展馆分为火箭技术区、行星探索区和深空未来区，每个区域对应不同展品和主题说明。

3. **九个展示节点**  
   展品包括火箭、卫星、火星、土星、空间站、探测器、月面基地、太阳系轨道模型和黑洞引力井，内容足够支撑 2-5 分钟展示视频。

4. **展品信息查看**  
   用户靠近展品后按 `E` 键，可以查看展品名称、科普说明和 3I 设计说明。

5. **展品拾取与观察**  
   部分展品支持拾取到视角前方，用鼠标拖动旋转，用滚轮缩放，突破真实展馆中不能触摸展品的限制。

6. **导览路线与视频辅助**  
   场景地面设置了 1-5 导览节点，入口处设置视频录制建议牌，引导用户按顺序展示整体场景、三个展区和 3I 总结。

7. **背景音乐系统**  
   系统将指定 mp3 文件作为 BGM，在开始参观后自动播放，使展示视频更完整、更有氛围。

### 3.4 实现过程

1. **项目主题与功能规划**  
   确定作品主题为太空探索虚拟展览馆，规划三大展区、九个展示节点、第一人称移动、展品信息查看、展品拾取观察、导览路线和 3I 分析展板等功能。

2. **Unity 场景搭建**  
   在 Unity 中搭建展览馆空间，包括地面、墙壁、天花板、玻璃隔断、展区标识、欢迎墙、展台、路线标识和说明展板，使场景具备完整展馆结构。

3. **脚本系统实现**  
   使用 C# 编写 `FirstPersonController.cs` 实现玩家移动和视角控制；使用 `GameManager.cs` 管理开始界面、暂停状态、信息面板和鼠标锁定；使用 `ExhibitBase.cs` 和 `ExhibitPickup.cs` 实现展品交互。

4. **展品模型生成**  
   使用 Unity 基础几何体组合生成展品模型，例如圆柱体组成火箭，立方体组成卫星太阳能板，球体组成行星，半透明圆盘表现土星环和黑洞吸积盘。

5. **UI 与交互反馈实现**  
   使用 Canvas 制作开始界面、信息面板、提示条和暂停菜单。用户靠近展品时显示提示，按键后显示展品说明。

6. **BGM 配置**  
   将 `maksymmalko-space-space-galaxy-universe-music-301239.mp3` 文件设置为背景音乐，开始参观后自动播放。

7. **测试与优化**  
   对中文字体、文本方向、展区文字摆放、信息面板关闭、`E` 键交互、灯光图标、音频播放和视频录制路线进行了多轮测试与优化。

## 四、3I 分析

### 4.1 Immersion 沉浸性

本作品通过完整展馆空间和第一人称视角增强沉浸性。用户进入场景后看到的是有墙壁、天花板、展台、展区标识和导览路线的封闭式虚拟展馆，而不是单纯漂浮在空中的模型。深色墙面、星空装饰、发光路线和背景音乐共同营造出太空科普展馆的氛围。

第一人称视角使用户能够像真实参观者一样在空间中移动和转头观察。地面 1-5 导览路线进一步强化了“正在参观一个展览”的体验，使用户能够按照空间提示逐步浏览展区。

### 4.2 Interaction 交互性

作品提供了多层交互方式。用户可以使用键盘移动、使用鼠标观察环境，可以靠近展品按 `E` 键查看详细信息。对于可拾取展品，用户可以将展品拿到眼前，通过鼠标拖动旋转、滚轮缩放来观察细节。

这些交互会直接改变系统状态：信息面板会显示或隐藏，展品会从展台移动到用户视角前方，提示 UI 会根据用户距离变化而出现或消失。系统对用户行为有明确反馈，因此不是静态展示，而是一个可操作的三维交互系统。

### 4.3 Imagination 构想性 / 想象性

太空探索本身具有强烈的想象性。现实中，普通用户难以近距离接触火箭、空间站、黑洞、深空探测器和月面基地，而虚拟展览馆可以把这些场景集中到一个可参观空间中。

本作品不仅展示已有航天成果，也加入了月面基地、黑洞引力井等未来或极端宇宙场景。用户可以在同一个虚拟空间中从近地航天走向深空未来，这种体验在现实展馆中成本很高，在虚拟空间中却可以低成本实现。因此，本作品体现了 VR 在科普教育、未来展览和远程学习中的应用想象。

## 五、展示视频录制设计

根据作业要求，展示视频建议时长为 2-5 分钟。本项目可按以下路线录制：

| 时间段 | 展示内容 |
|--------|----------|
| 0:00-0:25 | 展示开始界面，进入展馆，说明项目名称和主题 |
| 0:25-1:10 | 参观火箭技术区，展示火箭、卫星、月面基地，并演示 `E` 键交互 |
| 1:10-1:55 | 参观行星探索区，展示火星、土星和太阳系轨道模型 |
| 1:55-2:40 | 参观深空未来区，展示空间站、深空探测器和黑洞引力井 |
| 2:40-3:20 | 拍摄 3I 分析展板，说明沉浸性、交互性和构想性 |
| 3:20-3:40 | 总结作品亮点、不足和未来改进方向 |

按照该路线录制，视频能够自然达到 3 分钟左右，并覆盖作品运行效果、用户交互过程、核心功能和 3I 说明。

### 6.2 收获与体会

通过完成本作品，我们对虚拟现实系统的 3I 特征有了更具体的理解。沉浸性需要依靠空间结构、视角、氛围和导览路线共同建立；交互性需要用户操作后系统产生明确反馈；构想性则体现在虚拟空间能够呈现现实中难以到达或难以体验的内容。

同时，我们也熟悉了 Unity 三维项目的基本开发流程，包括场景搭建、玩家控制、碰撞检测、UI 制作、展品状态管理、音频配置和反复调试。项目从最初的简单展品陈列逐步优化为具有导览路线和展示节奏的虚拟展馆，也让我们意识到展示设计对于课程作业视频非常重要。

### 6.3 不足与改进方向

1. **美术精度仍可提升**  
   当前展品主要由 Unity 基础几何体组合生成，结构清晰但精细度有限。后续可以使用 Blender 或 Asset Store 高质量模型提升真实感。

2. **真实 VR 设备适配尚未完成**  
   当前使用键盘鼠标模拟体验。后续可以接入 OpenXR、SteamVR 或 Meta Quest，将视角映射到头显，将交互映射到手柄。

3. **多人互动功能不足**  
   当前是单人参观模式。未来可以接入网络框架，实现多人同时参观和语音讲解。

4. **展品内容仍可扩展**  
   当前已有 9 个展示节点，能够满足作业展示需要。后续可以设计动态展品加载系统，支持更多主题展览。

## 七、结论

本项目完成了一个可以运行、可以交互、可以录制展示视频的太空探索虚拟展览馆。作品通过第一人称漫游、三大展区、九个展示节点、展品信息面板、拾取观察、导览路线、3I 分析展板和背景音乐，较完整地体现了虚拟现实作品的沉浸性、交互性和构想性。

虽然当前版本仍是 PC 端模拟体验，但其结构已经具备进一步扩展到真实 VR 平台的基础。整体而言，本作品满足 Bonus 作业中“完成一个小型虚拟现实或交互式三维作品，并通过视频和报告展示”的要求。
