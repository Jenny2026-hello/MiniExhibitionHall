## 第一章：项目概述

### 1.1 项目简介

**MiniExhibitionHall（迷你交互展览馆）** 是一个基于 Unity 3D 引擎的第一人称虚拟展览体验项目。用户以第一人称视角在虚拟展览空间中自由行走，靠近展品查看详情，拾取展品进行360度观察，享受沉浸式的观展体验。

### 1.2 设计理念

本项目围绕虚拟现实的 **3I 核心特征** 进行设计：

| 特征 | 设计体现 |
|------|----------|
| **Immersion 沉浸性** | 完整的3D封闭空间 + 环境音效 + 第一人称视角，让用户感觉"走进了"一个展览馆 |
| **Interaction 交互性** | 自由走动、靠近展品触发展示、拾取物品旋转观察、鼠标操作实时反馈 |
| **Imagination 构想性** | 创造现实中无法实现的展览体验——展品可以漂浮旋转、可瞬间切换主题区域 |

### 1.3 技术选型理由

选择 Unity 的理由：
- 内置完善的3D物理和光照系统，适合构建沉浸式场景
- Asset Store 有大量免费3D资源，降低建模成本
- C# 脚本系统成熟，学习曲线平缓
- 可以直接打包为 PC 可执行程序，演示方便
- 运行效果流畅，视觉效果优于 Web 方案

## 第二章：系统架构

### 2.1 整体架构

```
┌─────────────────────────────────────────────────┐
│                   GameManager                     │
│            (全局状态管理 / UI控制)                  │
├─────────────────────────────────────────────────┤
│  ┌───────────┐  ┌───────────┐  ┌─────────────┐  │
│  │  Player   │  │ Exhibition│  │    Audio     │  │
│  │  Controller│  │  System   │  │   Manager    │  │
│  │           │  │           │  │             │  │
│  │ · 移动    │  │ · 展品基类 │  │ · BGM播放   │  │
│  │ · 视角    │  │ · 拾取系统 │  │ · 音效管理  │  │
│  │ · 重力    │  │ · 区域触发器│  │ · 环境音    │  │
│  └───────────┘  └───────────┘  └─────────────┘  │
│                                                  │
│  ┌───────────────────────────────────────────┐   │
│  │              UI System                     │   │
│  │  · 开始面板  · 信息面板  · 交互提示          │   │
│  │  · Billboard(始终面朝相机)                  │   │
│  └───────────────────────────────────────────┘   │
└─────────────────────────────────────────────────┘
```

### 2.2 脚本职责说明

#### GameManager.cs — 全局管理器（单例模式）
- 管理游戏整体状态（开始、运行中、暂停）
- 控制UI面板的显示/隐藏（开始面板、暂停菜单、展品信息面板）
- 维护鼠标锁定状态
- 提供全局接口供其他脚本调用

#### FirstPersonController.cs — 第一人称控制器
- 监听 WASD 输入实现移动
- 监听鼠标移动实现视角旋转
- 处理重力与地面检测（使用 CharacterController）
- Shift 加速奔跑功能

#### ExhibitBase.cs — 展品基类
- 通过 Trigger Collider 检测玩家是否靠近
- 在 Inspector 中可配置展品名称、描述、3I标签
- 靠近时显示交互提示，按 E 触发交互
- 支持自动旋转展示效果
- 支持配置专属音效

#### ExhibitPickup.cs — 可拾取展品（继承 ExhibitBase）
- 按 E 将展品挂载到相机下，实现"拿起"效果
- 鼠标左键拖拽旋转观察
- 鼠标滚轮缩放
- 再次按 E 放回原位

#### ZoneTrigger.cs — 区域触发器
- 玩家进入特定区域时触发事件
- 可用于切换展区、播放区域介绍
- 支持只触发一次或多次触发

#### AudioManager.cs — 音频管理器（单例模式）
- BGM 循环播放
- SFX 一次性播放（可叠加）
- 环境音独立通道
- 支持音量调节

#### Billboard.cs — 始终面向相机
- 使 UI 元素（标签、指示牌）始终朝向玩家
- 在 LateUpdate 中执行，保证渲染正确

### 2.3 数据流

```
用户输入(键盘/鼠标)
    │
    ▼
FirstPersonController ──→ 移动/视角更新
    │
    ▼
玩家位置变化 ──→ 触发 Collider
    │
    ▼
ExhibitBase/ZoneTrigger ──→ 检测交互
    │
    ▼
GameManager ──→ 更新UI / 显示信息
    │
    ▼
AudioManager ──→ 播放音效反馈
```

## 第三章：场景设计建议

### 3.1 推荐展览主题（5选1）

| 编号 | 主题 | 展品数量 | 难度 | 亮点 |
|------|------|----------|------|------|
| 1 | **古代文明展** | 4-6件 | ★★☆ | 兵马俑、青铜器、古钱币——国风有辨识度 |
| 2 | **太空探索展** | 4-6件 | ★★☆ | 火箭、行星、太空舱——3D模型简单 |
| 3 | **科技发展史** | 4-6件 | ★★☆ | 电脑、手机、AI芯片——与你专业相关 |
| 4 | **海洋生物展** | 4-6件 | ★★★ | 鱼类、珊瑚、深海生物——视觉效果好 |
| 5 | **未来城市展** | 3-5件 | ★★☆ | 建筑模型、交通工具、能源装置 |

**我的推荐**：选"太空探索展"或"科技发展史"——3D模型几何形状简单，用 Cube/Sphere/Cylinder 就能拼出不错的模型，而且与你的专业背景契合，报告好写。

### 3.3 光照设计

| 灯光类型 | 用途 | 设置建议 |
|----------|------|----------|
| Directional Light | 模拟天窗自然光 | Intensity: 0.5, Soft Shadows |
| Point Light | 重点照亮展品 | Range: 5, Color: 暖黄(#FFF3E0) |
| Spot Light | 聚焦核心展品 | Spot Angle: 45, Intensity: 1.5 |
| Ambient Light | 全局基础照明 | 通过 Lighting 窗口设置 |

## 附录：常见问题

**Q: 没有3D建模基础怎么办？**
A: Unity 内置的 Cube、Sphere、Cylinder 等基本几何体 + 调整颜色，就能做出不错的抽象风格展品。Asset Store 也有很多免费模型可以直接拖入使用。

**Q: 没有 VR 设备能展示3I特征吗？**
A: 可以。在报告中说明：本系统目前基于PC模拟VR体验，若接入VR设备，可将第一人称视角映射到HMD头显，将WASD移动映射到手柄摇杆行走，交互效果将成倍增强。

**Q: 展示视频怎么录？**
A: 用 OBS Studio（免费）录制屏幕 + 旁白解说。Windows 10/11 自带的 Xbox Game Bar (Win+G) 也能录制。
