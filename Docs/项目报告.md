## 二、项目创意与目标

### 2.1 为什么想做这个作品

本项目选择“太空探索虚拟展览馆”作为主题，是因为太空科技本身具有很强的空间感、探索感和想象空间。传统课堂或普通图片展示往往只能让用户被动观看火箭、行星、空间站等内容，而虚拟现实和三维交互可以让用户以第一人称方式进入一个完整的太空科普展馆，在场景中移动、观察、靠近展品并触发说明。

因此，本作品希望探索虚拟现实技术在科普教育和数字展览中的应用价值：通过一个可以运行、可以交互、可以录制展示的视频作品，让用户不只是“看见”太空展品，而是像进入真实展览馆一样主动参观和理解展品背后的知识。

### 2.2 希望为用户提供什么样的虚拟体验

本作品希望为用户提供以下体验：

1. **身临其境的虚拟观展体验**：用户从第一人称视角进入展馆，可以在有墙壁、地面、天花板、星空装饰、导览路线和展区标识的空间中自由移动。
2. **可交互的数字展品体验**：用户靠近展品后可以按 `E` 键查看信息说明，部分展品还可以被拿起、旋转和缩放观察。
3. **有主题结构的科普体验**：展馆划分为“火箭技术区”“行星探索区”“深空未来区”三个区域，并设置了 9 个展示节点，包括长征火箭、卫星、火星、土星、空间站、深空探测器、月面基地、太阳系轨道模型和黑洞引力井。
4. **适合展示视频录制的参观流程**：场景中加入了地面 1-5 导览路线、3I 分析墙和视频录制建议牌，使作品能够自然展示 2-5 分钟，而不是只停留在简单模型陈列。

## 三、系统设计与实现过程

### 3.1 系统概述

本系统是一个基于 Unity 引擎实现的第一人称虚拟展览馆。用户通过键盘和鼠标进行移动、视角控制和展品交互。系统通过三维场景、可交互展品、信息面板、导览路线和 3I 分析展板，构成一个完整的太空科普型虚拟现实作品。

虽然当前版本运行在普通 Windows PC 上，但其交互逻辑可以进一步映射到真实 VR 环境中：第一人称摄像机可以对应 VR 头显视角，键盘移动可以替换为手柄摇杆移动，`E` 键交互可以替换为手柄扳机键或射线选择。

### 3.2 输入与输出

#### 输入（用户操作）

| 操作 | 系统响应 |
|------|----------|
| `WASD` 键 | 用户在展馆中前后左右移动 |
| `Shift` 键 | 加速移动，用于快速浏览不同展区 |
| 鼠标移动 | 控制第一人称视角旋转 |
| 靠近展品后按 `E` | 查看展品信息，弹出说明面板 |
| 可拾取展品附近按 `E` | 将展品拿到镜头前观察 |
| 拿起展品后按住鼠标左键拖动 | 旋转手中的展品，观察结构细节 |
| 鼠标滚轮 | 调整手中展品的远近和观察尺度 |
| 信息面板打开时点击鼠标 / `Esc` / `Space` / `Enter` | 关闭信息面板 |
| `Esc` 键 | 打开或关闭暂停菜单 |

#### 输出（系统反馈）

- **视觉反馈**：展品模型自动旋转、地面导览路线、展区标识、3I 分析展板、开始界面、提示条、信息面板。
- **交互反馈**：靠近展品时出现交互提示；按键后展品信息面板显示；可拾取展品会移动到用户视角前方。
- **空间反馈**：用户进入不同区域时可以根据展区标识、地面路线和展品主题判断当前位置与参观顺序。

当前版本为了避免 Unity 编辑器中灯光和音频 Gizmo 图标干扰画面，弱化了运行时音频对象和真实 Light 对象的使用，主要通过环境光、材质颜色、发光材质和空间布局来营造氛围。

### 3.3 主要功能

1. **第一人称自由漫游**  
   用户可以在展馆内自由移动，通过鼠标环顾四周，模拟真实参观者在展览馆中行走和观察的体验。

2. **三大主题展区**  
   展馆分为“火箭技术区”“行星探索区”“深空未来区”，每个展区有不同颜色的视觉标识和对应展品。

3. **九个展示节点**  
   作品包含长征火箭模型、人造卫星模型、火星、土星、国际空间站模型、深空探测器模型、月面基地沙盘、太阳系轨道模型、黑洞引力井，共 9 个展示节点，使内容足够支撑 2-5 分钟展示视频。

4. **展品信息查看**  
   用户靠近展品后按 `E` 键，可以查看展品名称、说明文字，以及该展品在沉浸性、交互性、构想性方面的设计说明。

5. **展品拾取与观察**  
   部分展品支持拾取。用户可以将展品拿到眼前，用鼠标拖动旋转，用滚轮缩放，突破真实展览中“只能远观不能触摸”的限制。

6. **导览路线与视频引导**  
   场景地面设置了 1-5 的导览节点，入口处设置了视频录制建议牌，用户可以按照“入口展示 — 火箭区 — 行星区 — 深空区 — 3I 总结”的顺序录制视频。

7. **3I 分析展板**  
   展馆内设置了 Immersion、Interaction、Imagination 三块分析展板，便于在展示视频中直接说明作品如何体现虚拟现实系统的核心特征。

### 3.4 实现过程

1. **场景搭建**  
   使用 Unity 基础几何体搭建展览馆空间，包括地面、墙壁、天花板、玻璃隔断、立柱、星空装饰和欢迎墙。场景采用封闭式展馆结构，使用户进入后有明确的空间边界和参观方向。

2. **玩家控制系统**  
   使用 `FirstPersonController.cs` 实现第一人称移动与鼠标视角控制。玩家对象包含 `CharacterController`，能够在展馆空间中平滑移动。

3. **展品生成系统**  
   使用 `SpaceExhibitionBuilder.cs` 在 Unity 编辑器中一键生成完整场景。展品模型主要由 Unity 基础几何体组合而成，例如圆柱体组成火箭、立方体组成太阳能板、球体组成行星、圆盘表现土星环和黑洞吸积盘。

4. **展品交互系统**  
   通过 `ExhibitBase.cs` 实现展品的基础交互逻辑：检测玩家距离、显示提示、按 `E` 触发信息面板。通过 `ExhibitPickup.cs` 实现可拾取展品的拿起、放回、旋转和缩放。

5. **UI 系统**  
   使用 Unity Canvas 制作开始面板、信息面板、提示条和暂停菜单。`GameManager.cs` 管理开始、暂停、信息显示、信息关闭、鼠标锁定等全局状态。

6. **导览与展示优化**  
   为了满足最终提交中 2-5 分钟展示视频的要求，增加了地面 1-5 导览路线、视频录制建议牌和 3I 分析展板，使作品展示更有节奏和逻辑。

7. **调试与优化**  
   开发过程中修复了中文字体显示模糊、文本乱码、展区文字方向错误、信息面板关闭无效、`E` 键交互冲突、灯泡状 Gizmo 图标过多等问题。最终版本尽量减少编辑器图标干扰，使录制画面更干净。

## 四、3I 分析

### 4.1 Immersion 沉浸性

本作品通过以下设计增强用户进入虚拟环境的感觉：

1. **完整的三维展馆空间**  
   场景具有地面、墙壁、天花板、玻璃隔断、展区标识和欢迎墙，用户不是站在空白空间中看模型，而是进入一个具有空间结构的虚拟展览馆。

2. **第一人称视角**  
   用户通过第一人称摄像机观察世界，移动和转头都由自己控制，更接近真实参观体验，也更符合虚拟现实系统中“进入环境”的核心特征。

3. **太空主题视觉氛围**  
   场景中设置了星空装饰、深色墙面、发光路线、发光展台和不同展区颜色，使整个空间具有太空探索的视觉风格。

4. **展区式空间组织**  
   火箭技术区、行星探索区和深空未来区分别对应不同展品内容，用户在空间中移动时能够感受到从近地航天、行星探索到深空未来的主题递进。

5. **导览路线增强参观感**  
   地面 1-5 导览路线让用户像在真实展馆中跟随路线参观一样，增强了“正在游览一个完整展览”的体验。

### 4.2 Interaction 交互性

本作品不仅是静态场景，还提供了多种用户交互：

1. **移动与观察交互**  
   用户通过 `WASD` 控制自身位置，通过鼠标控制视角，参观顺序和观察角度由用户决定。

2. **靠近触发提示**  
   当用户进入展品交互范围时，系统会显示“按 E 查看详情”或“按 E 拿起观察”等提示，形成即时反馈。

3. **信息面板交互**  
   用户按 `E` 后可以打开展品信息面板，阅读展品介绍和 3I 说明。信息面板可以通过按钮、鼠标点击或快捷键关闭。

4. **拾取与观察交互**  
   部分展品支持被拿起到镜头前，用户可以通过鼠标拖动旋转、滚轮缩放。这种交互突破了真实展览中展品不能触碰的限制，是本作品的核心互动亮点。

5. **区域导览交互**  
   场景中设置了三个区域触发器，用户进入不同展区时，系统可以识别用户所在区域并输出对应导览信息。

用户的行为会直接影响系统状态：例如打开或关闭 UI、改变展品位置、旋转展品、切换暂停状态等。这些反馈使作品不只是一个可观看的模型集合，而是一个可以操作和探索的交互系统。

### 4.3 Imagination 构想性 / 想象性

本作品的构想性主要体现在以下方面：

1. **把现实中难以接近的主题变成可参观空间**  
   太空探索、黑洞、深空探测、月面基地等内容在现实中很难被普通用户亲身体验。本作品通过虚拟展馆将这些内容转化为可以进入、观察和互动的三维场景。

2. **突破真实展览的物理限制**  
   真实博物馆中的航天模型通常只能远距离观看，而本作品允许用户拿起部分展品、旋转观察、缩放细节，提供了传统展览无法实现的体验方式。

3. **科普教育与虚拟空间结合**  
   展品不仅是模型，还配有说明文字和 3I 分析。用户在参观过程中可以同时获得视觉体验和知识信息，使作品具有教育展示功能。

4. **未来应用想象**  
   该项目可以扩展为更完整的虚拟科普平台，例如加入 VR 头显支持、多人在线参观、讲解员模式、互动问答、任务式学习路线等。它展示了未来教育和文化传播不必局限于物理场馆的可能性。

## 五、展示视频设计

根据作业要求，展示视频建议时长为 2-5 分钟。本项目在场景中加入了专门的视频录制路线，建议视频按以下结构录制：

| 时间段 | 内容 |
|--------|------|
| 0:00-0:25 | 展示开始界面，进入展馆，说明作品名称和主题 |
| 0:25-1:10 | 沿导览路线进入火箭技术区，展示长征火箭、人造卫星、月面基地，并演示 `E` 键交互 |
| 1:10-1:55 | 进入行星探索区，展示火星、土星、太阳系轨道模型，说明沉浸式观察体验 |
| 1:55-2:40 | 进入深空未来区，展示空间站、深空探测器、黑洞引力井，说明构想性 |
| 2:40-3:20 | 拍摄 3I 分析展板，总结 Immersion、Interaction、Imagination |
| 3:20-3:40 | 简要总结作品意义、不足和未来改进方向 |

通过该路线，视频能够自然达到 3 分钟左右，并覆盖作业要求中的运行效果、用户交互过程、核心功能和 3I 说明。

### 6.2 收获与体会

通过完成本作品，我主要获得了以下收获：

1. **对虚拟现实 3I 特征有了实践理解**  
   沉浸性并不是简单地把模型放进 3D 场景，而需要空间结构、视觉风格、第一人称视角和参观节奏共同支撑。交互性也不只是“能点击”，而是用户操作后系统需要有明确反馈。

2. **掌握了 Unity 三维交互项目的基本流程**  
   项目涉及场景搭建、玩家控制、碰撞检测、UI 面板、脚本通信、展品状态切换、编辑器工具生成场景等多个环节。

3. **体会到展示设计的重要性**  
   一开始场景内容较少，难以录制 2-5 分钟视频。后来通过增加导览路线、展品数量和 3I 展板，作品才更像一个完整项目，而不是简单模型展示。

4. **提升了调试和优化能力**  
   开发过程中遇到了字体模糊、乱码、文字方向错误、交互无效、灯光 Gizmo 干扰等问题。通过反复测试和修改，逐步提升了作品的可展示性。

### 6.3 不足与改进方向

1. **美术精度仍有限**  
   当前展品主要使用 Unity 基础几何体组合生成，虽然结构清楚，但与真实高精度模型相比仍有差距。未来可以引入更高质量的模型资源或使用 Blender 制作更精细的展品。

2. **音频与环境氛围还可加强**  
   当前版本为了避免录制画面中出现编辑器图标，弱化了音频对象。后续可以在正式构建版本中重新加入背景音乐、环境音和交互音效。

3. **尚未接入真实 VR 设备**  
   当前作品运行在 PC 上，通过键盘鼠标模拟 VR 参观。未来可以接入 SteamVR、OpenXR 或 Meta Quest，将头显视角和手柄交互映射到系统中。

4. **缺少多人协同体验**  
   当前是单人参观模式。未来可以接入 Photon、Mirror 等网络框架，支持多人同时进入展馆参观、交流和协作学习。

5. **展品内容可以继续扩展**  
   当前已有 9 个展示节点，能够满足 Bonus 作业展示需求。未来可以增加展品加载系统，让展馆支持动态扩展和更新。

## 七、结论

本项目完成了一个可以运行、可以交互、可以展示的太空探索虚拟展览馆。作品通过第一人称漫游、三大主题展区、九个展示节点、展品信息面板、拾取观察、导览路线和 3I 分析展板，较完整地体现了虚拟现实系统的沉浸性、交互性和构想性。

虽然当前版本仍是 PC 端模拟体验，但其结构已经具备进一步扩展到真实 VR 平台的基础。整体而言，本作品达到了 Bonus 作业“完成一个小型虚拟现实或交互式三维作品，并通过视频和报告清楚展示”的要求。
